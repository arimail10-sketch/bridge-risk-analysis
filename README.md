# 🌉 Bridge Risk Analysis Dashboard

データサイエンスの個人学習記録です。
オープンデータから東京23区の橋梁データをもとに、**交通量・老朽化・迂回路長**の観点から、
補修・補強の優先度設定を試行した結果を可視化するダッシュボードです。

**🔗 アプリはこちら**: https://bridge-risk-analysis-kwb44vmgfauxjuehd5ch3h.streamlit.app/

---

## 📌 このプロジェクトについて

橋梁の老朽化対策は全国的な課題ですが、限られた予算の中でどの橋から手をつけるべきか、
という優先順位づけには「橋そのものの劣化度」だけでなく、
「その橋が通行止めになった場合に地域にどれだけ影響が出るか」という視点も重要だと考えました。

本プロジェクトでは、行政のオープンデータと地図データを組み合わせることで、
以下の3つの観点から橋梁の優先度を評価しています。

1. **構造的特徴**：架設年度、幅員、橋長などから、K-Meansクラスタリングによって6つのグループに分類
2. **交通量**：橋の周辺の24時間自動車交通量
3. **迂回路長**：その橋が通行止めになった場合に、迂回するために必要な追加距離

これらを統合し、独自の優先度スコアで橋梁をランキングしています。

## 🗺️ ダッシュボードでできること

- クラスター（橋の特徴グループ）や道路種別による絞り込み
- 優先度上位30件のみの表示
- 橋名を指定してのピンポイント検索・表示
- 地図上のクリックで、各橋の詳細情報（架設年度・交通量・迂回路長など）を確認

## 🔧 使用データ

- [国土交通データプラットフォーム](https://www.mlit-data.jp/)：橋梁データ（rsdb_bridge）、道路交通センサス（rtc_2021）
- [OpenStreetMap](https://www.openstreetmap.org/)：道路ネットワークデータ（`osmnx`経由で取得）

## 🛠️ 技術スタック

- Python (pandas, geopandas, scikit-learn, networkx, osmnx)
- Streamlit / streamlit-folium（ダッシュボード）

## ⚠️ 既知の課題

- 国土交通データプラットフォームの橋とOpenStreetMapで位置情報が一致していない橋がある
- 交通量はボロノイ分割の空間結合としたため路線の交通量と一致していない橋がある
- 迂回路計算が道路の進行方向を踏まえたものかまでは未検証
- 一部の橋梁で迂回路が計算できていないケースがある
